Stock Incremental Game — Full Pre-Production Design Document
Product Vision
This game is an incremental idle game themed around building a trading firm from a solo operation into a highly automated financial empire. The tone is clean, semi-serious, and slightly exaggerated. It should feel like a sleek finance dashboard with Bloomberg-terminal energy, while still allowing some stylized progression through promotions, upgraded desks, and automated trading infrastructure.
The player fantasy is:
start as a solo trader → hire Junior Traders → promote them into Senior Traders → unlock Trading Bots later as a prestigious automation layer → prestige into a larger financial reputation loop
The design goal is to keep the economy understandable, the progression satisfying, and the first full run substantial.
---
Genre Identity
This is an incremental idle game with a strong early active phase and a long-form first prestige cycle.
The intended structure is:
Clicker opening — the player manually clicks Make Trade for cash.
Human automation phase — the player hires Junior Traders.
Firm-building phase — the player promotes Juniors into Senior Traders.
Advanced automation phase — the player unlocks Trading Bots late in the run.
Prestige loop — the player gains permanent Reputation and repeats the loop faster and deeper.
The first run should not feel disposable. It should feel like a full progression arc.
---
Design Pillars
Readable finance fantasy
Currency, units, and upgrades should all feel like believable parts of a trading firm.
Late bot payoff
Trading Bots should arrive meaningfully later, after the player has spent a large portion of the run building human infrastructure.
No obsolete early units
Junior Traders stay relevant because they can be promoted into Senior Traders.
Prestige as meta progression
Prestige does not unlock the basic phases. It accelerates and extends them.
UI-first clarity
The interface should feel like a clean terminal, with sharp information hierarchy and minimal clutter.
---

First Prestige Cycle
60–90 minutes for a normal first run.
This means:
the first prestige is a significant milestone
bots arrive late enough to feel prestigious
the player spends real time in the human-firm phases before reaching advanced automation
This longer pacing creates a more deliberate, finance-sim style progression rather than a hyper-casual tapper.

Core Gameplay Loop
Click Make Trade to earn Cash.
Buy manual upgrades to improve click income.
Hire Junior Traders for passive income.
Upgrade Junior Traders.
Unlock Promotion Program and promote Juniors into Senior Traders.
Use human staff as the main mid-game economy.
Unlock Trading Bots much later.
Upgrade bots and stack global multipliers.
Reach a progression slowdown.
Prestige to gain Reputation.
Spend Reputation on permanent boosts and repeat the loop faster.

Core Progression Phases and Timing
Phase 1 — Solo Trader
Target duration: 0–8 minutes
The player is trading manually and learning the core loop.
Goals:
buy Better Terminal
buy first few click upgrades
save for first Junior Trader
Expected economy at this phase:
Cash range: $0 to $250
Click income: $1 to around $6
Passive income: $0 to around $2/sec
Player feeling:
“I’m doing this myself.”
---
Phase 2 — Junior Trader Desk
Target duration: 8–30 minutes
The player grows a small team and establishes passive income.
Goals:
buy multiple Junior Traders
improve Junior Trader efficiency
transition from mostly active to mixed active/passive play
Expected economy at this phase:
Cash range: $250 to $8,000
Click income: $6 to around $20
Passive income: $2/sec to around $60/sec
Player feeling:
“My desk is starting to function without me.”
---
Phase 3 — Senior Trader Firm Growth
Target duration: 30–60 minutes
This is the main pre-bot phase and should be the longest human-centered portion of the run.
Goals:
unlock Promotion Program
promote Juniors into Seniors
scale the firm through stronger human capital
stack key global multipliers
Expected economy at this phase:
Cash range: $8,000 to $80,000
Click income: $20 to around $60
Passive income: $60/sec to around $500/sec
Player feeling:
“I’m building a serious firm now.”
---
Phase 4 — Trading Bot Era
Target duration: 60–85 minutes
Bots should arrive late and feel expensive, aspirational, and transformational.
Goals:
unlock Algorithmic Trading
buy first Trading Bots
improve bot performance
push into late-run scaling
Expected economy at this phase:
Cash range: $80,000 to $500,000+
Click income: $60+
Passive income: $500/sec to several thousand/sec
Player feeling:
“I’ve crossed into institutional-scale automation.”
---
Phase 5 — Prestige Decision
Target duration: 80–90+ minutes
The player should feel the slowdown clearly and understand that prestige is now the optimal next move.
Expected economy at this phase:
Lifetime Cash Earned: roughly $750,000 to $1,500,000 on the first prestige cycle
Reputation gain should be visible and meaningful
Player feeling:
“I could keep pushing, but restarting now is clearly smarter.”
---
Pacing Rule for Trading Bots
Trading Bots should appear only after the player has spent the majority of the first run in human-led progression.
Rule
The player should spend roughly two thirds of the first run before bots become meaningfully available.
That means bots are not an early generator. They are a late milestone.

Core Formulas
Manual Income Formula
Cash per Click = Base Click Value × Manual Multipliers × Global Multipliers × Prestige Multipliers
Passive Income Formula
Cash per Second = (Junior Output Total + Senior Output Total + Bot Output Total) × Global Multipliers × Prestige Multipliers
Unit Cost Formula
Next Unit Cost = Base Cost × (Scaling Rate ^ Owned Count)
Recommended scaling:
Junior Trader: 1.18
Trading Bot: 1.20
Promotion Formula
Promotion is not a direct purchase of a new unit.
It is a transformation:
subtract 1 Junior Trader
add 1 Senior Trader
charge promotion cash cost
Global Multiplier Stacking
All global multipliers stack multiplicatively.
This keeps them meaningful throughout the run.
Prestige Multiplier Formula
Reputation should provide:
a direct permanent money boost
permanent meta-upgrades purchased with Reputation
Baseline direct prestige bonus recommendation:
Total Money Multiplier from Reputation = 1 + (0.05 × Reputation Spent or Earned milestone basis)
This exact value should be tuned in balancing, but the intent is that prestige is clearly powerful without trivializing the next run.

Offline Progress
The game should support offline progress with a cap, framed as capped trading hours.
Rule

Derived Economy Functions
These functions should live in `utils/economy.ts` and be pure.
1. Unit Cost Calculation
```ts
export function getScaledCost(baseCost: number, scaling: number, owned: number): number {
  return Math.floor(baseCost * Math.pow(scaling, owned));
}
```
2. Manual Income Calculation
```ts
export function getManualIncome(state: GameState): number {
  let value = 1;

  if (state.purchasedUpgrades['betterTerminal']) value = 2;
  if (state.purchasedUpgrades['hotkeyMacros']) value += 2;
  if (state.purchasedUpgrades['premiumDataFeed']) value *= 1.5;

  value *= getGlobalMultiplier(state);
  value *= getPrestigeMultiplier(state);

  return value;
}
```

```ts
export function getJuniorTraderIncome(state: GameState): number {

  return value;
}
```

4. Senior Trader Income Calculation

```ts
export function getSeniorTraderIncome(state: GameState): number {
```

5. Trading Bot Income Calculation
```ts
export function getTradingBotIncome(state: GameState): number {
```

6. Global Multiplier Calculation
```ts
export function getGlobalMultiplier(state: GameState): number {
  let multiplier = 1;

  if (state.purchasedUpgrades['tradeMultiplier']) multiplier *= 1.25;
  if (state.purchasedUpgrades['bullMarket']) multiplier *= 1.5;

  return multiplier;
}
```
7. Prestige Multiplier Calculation
```ts
export function getPrestigeMultiplier(state: GameState): number {
  const directReputationBonus = 1 + state.reputation * 0.05;
  const brandRecognitionRank = state.purchasedPrestigeUpgrades['brandRecognition'] || 0;
  const brandRecognitionBonus = 1 + brandRecognitionRank * 0.1;

  return directReputationBonus * brandRecognitionBonus;
}
```
8. Total Passive Income Calculation
```ts
export function getCashPerSecond(state: GameState): number {
  const juniorIncome = state.juniorTraderCount * getJuniorTraderIncome(state);
  const seniorIncome = state.seniorTraderCount * getSeniorTraderIncome(state);
  const botIncome = state.tradingBotCount * getTradingBotIncome(state);

  return (juniorIncome + seniorIncome + botIncome)
    * getGlobalMultiplier(state)
    * getPrestigeMultiplier(state);
}
```
---
Reputation Gain Formula
For MVP, use a simple, readable prestige formula.
Recommendation
```ts
export function getPrestigeGain(lifetimeCashEarned: number): number {
  if (lifetimeCashEarned < 1000000) return 0;
  return Math.floor(Math.sqrt(lifetimeCashEarned / 1000000));
}
```
What this does
no prestige before $1,000,000 lifetime cash
1 Reputation around first prestige threshold
scales gradually instead of exploding too fast
This formula is intentionally conservative for a long first-run game.

This should live in `utils/offlineProgress.ts`.
Rule
determine elapsed time since last save

MVP Function
```ts
export function getOfflineSecondsApplied(lastSaveTimestamp: number, now: number): number {
  const elapsedSeconds = Math.floor((now - lastSaveTimestamp) / 1000);
  return Math.min(elapsedSeconds, 28800);
}
```

Number Formatting Logic
This should live in `utils/formatting.ts`.
Rule
use comma formatting below 1,000,000
switch to short format at or above 1,000,000
Example Function
```ts
export function formatNumber(value: number): string {
  if (value < 1000000) {
    return Math.floor(value).toLocaleString();
  }

  if (value < 1000000000) {
    return `${(value / 1000000).toFixed(1)}M`;
  }

  return `${(value / 1000000000).toFixed(1)}B`;
}
```

Final Design Summary
This game is a finance-themed incremental idle game with a clean terminal presentation and a deliberate long-form first run.
The player begins alone, builds a human trading desk, promotes early hires into stronger Senior Traders, unlocks Trading Bots late as aspirational automation, and then prestiges into Reputation, which increases money gained overall and unlocks permanent meta-progression.
The first prestige should come only after the player has fully experienced the human-led phases and reached the bot phase. Prestige accelerates and extends the loop rather than replacing the core progression.
The MVP should launch with the full core arc in place:
manual trading → Junior Traders → Senior Traders → Trading Bots → Reputation prestige
That gives the project a strong, buildable foundation while leaving clear room for future late-game expansion.
